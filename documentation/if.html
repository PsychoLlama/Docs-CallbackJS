If

Callback decisions

Say your callback should only be invoked if a condition is met. Instead of cluttering your callback with tests, you can use if to separate the logic. if takes a function to execute before your callback is invoked, and if it returns a falsey value, your callback won't run. 

new Callback( function )
.if( function(user) {
  var isObj, hasPass;
  
  isObj = typeof user === 'object'
  hasPass = typeof user.password === 'string'
  
  return isObj && hasPass
})

The example above checks the validity of user, returning true if the conditions are met. This keeps the testing logic separate from the callback logic, allowing your code to stay clean and tidy.
