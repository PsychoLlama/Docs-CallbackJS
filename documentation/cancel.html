Cancel

Removing a callback

Immediate

To prevent a callback from firing, you can call .cancel() on it. This allows you to attach your function to events without invoking it. 

new Callback()
.when( input, 'change' )
.cancel()

The callback in the example will not fire until a renew method is called on it.

Event driven

.cancel() takes arguments. In fact, under the hood, cancel uses .when and can take any event that .when can. This becomes very powerful when you need to describe a callback that expires. Here's an example: 

new Callback( function )
.when( button, "click" )
.cancel( request, "load" )
.cancel( 5000 )

This callback invokes when a button is clicked, but the event expires either when your request loads or in five seconds, whichever happens first.
