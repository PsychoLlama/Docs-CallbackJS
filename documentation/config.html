Config

Behavior customization

config lets you change advanced details on how your function is invoked. Generally you won't need it, but it's there if you do. It takes an object of configuration variables as an argument.

Arguments

By default, the event object is passed to your callback. If you want to override this, you can do that in the configuration object... 

new Callback()
.config({
  // one argument
  pass: argVar
  
  // multiple arguments
  pass: [arg1, arg2...]
})

"this"

You can know for sure what the value of this is by setting it in the config object. Here's an example: 

new Callback()
.config({
  // remember to wrap "this" in quotes
  "this": friendlyPotato
})

In the example above, when your callback is invoked, this will be equal to friendlyPotato.

Although this feature is handy, I recommend only using it for troubleshooting. Others may make assumptions as to the value of this when reading your code.

Others

The other configuration variables have methods of their own, and to preserve the semantics and explicitness of your code, I recommend using them. That said, if you feel the config method is more readable, have at it. Here are the properties available to you:

cancelled

Whether or not to execute this callback when it is invoked. Setting this value as true is similar to calling the cancel method. 

config({
  cancelled: boolean
})

conditional

Assign a function to run before the callback is invoked, deciding whether or not it should be. Essentially the same as the if method. 

config({
  conditional: function
})

error

Give it a function to call if your callback throws an error. Does the same thing as the catch method. 

config({
  error: handler
})
