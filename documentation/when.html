When

Event binding

DOM events

After declaring a callback, you can bind it to a limitless number of events using "when". 

new Callback(function)
.when( target, "event" )

In this case, when takes an element as a first parameter and an event to search for in the second. This might look like ( document, "click" ).

Promises

CallbackJS checks the target "ready state" of your load events. If the document/request etc. has already ended, your callback is fired immediately.

Timer events

when also takes time as an argument. Pass your time in milliseconds and the callback will execute when it ends. Also, if you want it to repeat, you can pass true afterwards. 

new Callback()
// call once after three seconds
.when(3000)

// repeat every five seconds
.when(5000, true)

Custom events

The crowning feature of CallbackJS is custom events. Using them with when is as simple as passing a string as your custom event name. 

new Callback()
.when( "squirrel moves" )
.when( "drone starts" )
